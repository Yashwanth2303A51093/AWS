Deploy Windows Server on AWS EC2

A modern, visual guide to help you launch, configure, and access your Windows Server instance securely.

Step-by-Step Instructions

1. Log in to AWS Console

Navigate to console.aws.amazon.com and open the EC2 Dashboard.

2. Launch New EC2 Instance

Click Launch Instance and enter a descriptive name like Windows-Server-Test.

3. Choose Windows AMI

  • Choose Windows Server 2019 Base or 2022 Base.
  • Ensure it includes RDP support.

4. Select Instance Type

Recommended: t2.medium (2 vCPUs, 4GB RAM). Select Free-Tier eligible options if needed.

5. Create or Choose Key Pair

  • Select RSA key format.
  • Download and keep your .pem file secure.

6. Configure Security Group

Allow RDP (TCP port 3389) access from My IP only. Avoid 0.0.0.0/0.

7. Launch & Retrieve Password

  • Launch instance and wait until running.
  • Use your .pem file to decrypt the Administrator password.

8. Connect via RDP

Use Remote Desktop with the public IP and decrypted credentials.

Security & Best Practices

  • Use IAM roles over hardcoded credentials.
  • Always restrict RDP access to your IP.
  • Assign Elastic IP for static access.
  • Update Windows immediately after first login.

Cleanup After Use

  • Terminate the instance when done to avoid charges.
  • Release Elastic IPs and delete unused key pairs.
  • Remove unattached volumes to save storage costs.
